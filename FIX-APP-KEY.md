# Исправление ошибки MissingAppKeyException

## Проблема

Ошибка: `No application encryption key has been specified.`

Это означает, что в Railway не установлена переменная окружения `APP_KEY`.

## Решение

### Вариант 1: Сгенерировать ключ локально и добавить в Railway

1. Локально выполните:
```bash
php artisan key:generate --show
```

2. Скопируйте сгенерированный ключ (начинается с `base64:`)

3. В Railway → Variables добавьте:
   - **Key:** `APP_KEY`
   - **Value:** `base64:скопированный-ключ`

### Вариант 2: Сгенерировать ключ на сервере

Через Railway Shell выполните:

```bash
php artisan key:generate --force
```

Это автоматически добавит ключ в `.env` файл.

### Вариант 3: Использовать команду в Railway

В настройках деплоя Railway добавьте команду после `composer install`:

```bash
php artisan key:generate --force || true
```

Или добавьте в скрипты деплоя.

## Проверка

После добавления `APP_KEY`:

1. Обновите страницу приложения
2. Ошибка должна исчезнуть
3. Должна открыться страница Laravel Welcome

## Важно

`APP_KEY` - критически важная переменная для Laravel. Без неё приложение не может работать, так как Laravel использует его для:
- Шифрования данных
- Сессий
- Cookies
- И других криптографических операций
