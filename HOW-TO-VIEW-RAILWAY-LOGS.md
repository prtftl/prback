# Как посмотреть логи в Railway

## Способ 1: Логи деплоя (Deployments)

### Шаг 1: Откройте Deployments

1. Railway → ваш проект
2. Откройте **Laravel сервис**
3. Перейдите на вкладку **"Deployments"**

### Шаг 2: Откройте последний деплой

1. Найдите последний деплой в списке (обычно самый верхний)
2. Нажмите на него, чтобы открыть детали

### Шаг 3: Просмотрите логи

1. В открывшемся окне будут логи деплоя
2. Прокрутите вниз, чтобы увидеть все логи
3. Ищите ошибки и предупреждения

### Что искать в логах деплоя:

**Если Nova устанавливается:**
```
Installing laravel/nova (5.7.6)
Publishing Nova Assets / Resources
Nova scaffolding installed successfully.
```

**Если есть ошибки:**
```
[RuntimeException] Missing or incorrect username / password combination
```
→ Проблема с `COMPOSER_AUTH`

```
Package laravel/nova not found
```
→ Nova не установилась

## Способ 2: Логи приложения (Metrics/Logs)

### Шаг 1: Откройте Metrics или Logs

1. Railway → ваш проект
2. Откройте **Laravel сервис**
3. Перейдите на вкладку **"Metrics"** или **"Logs"**

### Шаг 2: Просмотрите логи

1. В разделе Logs будут логи работающего приложения
2. Ищите ошибки, связанные с Nova

### Что искать в логах приложения:

**Ошибки 404:**
```
GET /nova 404
```

**Ошибки маршрутов:**
```
Route [nova.login] not defined
```

**Ошибки провайдеров:**
```
Class 'Laravel\Nova\Nova' not found
```

## Способ 3: Через веб-интерфейс Railway

### Шаг 1: Откройте сервис

1. Railway → ваш проект
2. Откройте **Laravel сервис**

### Шаг 2: Найдите раздел с логами

В зависимости от версии Railway, логи могут быть в:
- **"Deployments"** → последний деплой → "View Logs"
- **"Metrics"** → вкладка "Logs"
- **"Logs"** (отдельная вкладка)

## Что проверить в логах для диагностики Nova

### 1. Проверьте установку Nova

Ищите в логах деплоя:
```
Installing laravel/nova
Publishing Nova Assets
Nova scaffolding installed successfully.
```

**Если этого нет:**
- Nova не установилась
- Проверьте `COMPOSER_AUTH`

### 2. Проверьте ошибки при установке

Ищите ошибки:
```
[RuntimeException] Missing or incorrect username / password
```
→ Проблема с `COMPOSER_AUTH`

```
Package laravel/nova not found
```
→ Nova не установилась

### 3. Проверьте логи приложения

Ищите ошибки при обращении к `/nova`:
```
GET /nova 404
Route [nova.login] not defined
Class 'Laravel\Nova\Nova' not found
```

## Диагностика проблемы 404

### Шаг 1: Проверьте логи деплоя

1. Откройте последний деплой
2. Просмотрите логи
3. Ищите установку Nova

**Вопросы:**
- Установилась ли Nova? (ищите "Installing laravel/nova")
- Были ли ошибки при установке?
- Выполнилась ли команда `php artisan nova:install`?

### Шаг 2: Проверьте логи приложения

1. Откройте Metrics/Logs
2. Попробуйте открыть `/nova` в браузере
3. Смотрите логи в реальном времени

**Что искать:**
- Ошибки 404 при обращении к `/nova`
- Ошибки маршрутов
- Ошибки провайдеров

### Шаг 3: Проверьте переменные окружения

Railway → Laravel сервис → Variables:

**Обязательно должны быть:**
- `COMPOSER_AUTH` - для установки Nova
- `DB_CONNECTION=mysql` - для подключения к БД
- `DB_URL` - строка подключения к MySQL

## Быстрая диагностика

### Проверка 1: Установлена ли Nova?

В логах деплоя ищите:
```
Installing laravel/nova (5.7.6)
```

**Если нет:**
- Проверьте `COMPOSER_AUTH`
- Проверьте Build Command

### Проверка 2: Зарегистрированы ли маршруты?

В логах приложения при обращении к `/nova` ищите:
- 404 - маршруты не зарегистрированы
- Страница входа - все работает! ✅

### Проверка 3: Есть ли ошибки?

В логах ищите:
- `Class 'Laravel\Nova\Nova' not found` - Nova не установлена
- `Route [nova.login] not defined` - маршруты не зарегистрированы
- Ошибки подключения к БД - проблема с переменными БД

## Что делать после просмотра логов

### Если Nova не установилась:

1. Проверьте `COMPOSER_AUTH` в Railway Variables
2. Проверьте Build Command
3. Запустите новый деплой

### Если Nova установилась, но 404:

1. Проверьте логи приложения на ошибки
2. Проверьте, что `NovaServiceProvider` загружается
3. Очистите кеш (если возможно)

### Если есть ошибки в логах:

1. Скопируйте ошибку
2. Найдите решение в документации
3. Или сообщите мне - помогу исправить

## Резюме

**Что искать:**
- Установка Nova
- Ошибки при установке
- Ошибки при обращении к `/nova`

**После просмотра:**
