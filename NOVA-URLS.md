# Ссылки для доступа к Nova

## Основной путь

Согласно конфигурации в `config/nova.php`, Nova доступна по пути:

```
/nova
```

## Полные URL для вашего проекта

### Основные ссылки:

1. **Главная страница Nova (страница входа):**
   ```
   https://prback-production.up.railway.app/nova
   ```

2. **Страница входа (явно):**
   ```
   https://prback-production.up.railway.app/nova/login
   ```

3. **API маршруты Nova (не для прямого доступа):**
   ```
   https://prback-production.up.railway.app/nova-api/...
   ```

## Что должно открываться

### Если Nova установлена и работает:

**По адресу `/nova`:**
- Должна открыться страница входа в Nova
- Форма для входа с email и password
- Логотип Nova вверху

**По адресу `/nova/login`:**
- Та же страница входа

### Если Nova НЕ установлена:

**По адресу `/nova`:**
- 404 NOT FOUND
- Или ошибка Laravel

**По адресу `/nova/login`:**
- 404 NOT FOUND

## Проверка через браузер

### Шаг 1: Откройте основной путь

```
https://prback-production.up.railway.app/nova
```

**Ожидаемый результат:**
- ✅ **Страница входа Nova** - Nova установлена и работает!
- ❌ **404 NOT FOUND** - Nova не установлена или не зарегистрирована
- ❌ **500 Internal Server Error** - Проблема с конфигурацией

### Шаг 2: Попробуйте альтернативные пути

Если `/nova` не работает, попробуйте:

```
https://prback-production.up.railway.app/nova/login
```

## Изменение пути Nova

Если хотите изменить путь (например, на `/admin`):

1. Откройте `config/nova.php`
2. Измените:
   ```php
   'path' => '/admin',  // вместо '/nova'
   ```
3. Очистите кеш:
   ```bash
   php artisan config:clear
   ```

Тогда Nova будет доступна по:
```
https://prback-production.up.railway.app/admin
```

## Важно

- Путь `/nova` - это стандартный путь по умолчанию
- Если вы видите 404, это означает, что:
  - Nova не установлена на сервере
  - Или маршруты Nova не зарегистрированы
  - Или есть проблема с конфигурацией

## Диагностика

### Если `/nova` возвращает 404:

1. **Проверьте логи деплоя:**
   - Установлена ли Nova?
   - Есть ли ошибки при установке?

2. **Проверьте переменные окружения:**
   - Установлен ли `COMPOSER_AUTH`?
   - Установлен ли `NOVA_LICENSE_KEY`?

3. **Проверьте конфигурацию:**
   - Есть ли `config/nova.php`?
   - Правильно ли настроен `NovaServiceProvider`?
