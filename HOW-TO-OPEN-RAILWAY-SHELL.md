# Как открыть Shell/Terminal в Railway

## Пошаговая инструкция

### Шаг 1: Откройте Railway

1. Перейдите на сайт [railway.app](https://railway.app)
2. Войдите в свой аккаунт
3. Откройте ваш проект

### Шаг 2: Найдите ваш Laravel сервис

В списке сервисов найдите сервис с вашим Laravel приложением. Обычно он называется:
- По имени проекта
- Или "Web Service"
- Или "Laravel"

**Важно:** Нужен именно **Laravel сервис**, а не MySQL сервис!

### Шаг 3: Откройте сервис

Нажмите на ваш Laravel сервис, чтобы открыть его страницу.

### Шаг 4: Найдите вкладку "Shell" или "Terminal"

После открытия сервиса вы увидите вкладки в верхней части страницы:

- **Deployments** - история деплоев
- **Metrics** - метрики
- **Variables** - переменные окружения
- **Shell** или **Terminal** - ⭐ **ВОТ ЭТО НУЖНО!**
- **Settings** - настройки

Нажмите на вкладку **"Shell"** или **"Terminal"**.

### Шаг 5: Откройте Shell

После нажатия на вкладку откроется терминал внизу страницы или в отдельном окне.

Вы увидите командную строку, похожую на:
```
$ 
```

Готово! Теперь вы можете выполнять команды.

## Альтернативный способ: Через Deployments

Если не можете найти вкладку Shell:

1. Откройте вкладку **"Deployments"**
2. Найдите последний деплой
3. Рядом с деплоем может быть кнопка **"Shell"** или **"Terminal"**
4. Нажмите на неё

## Что делать после открытия Shell

После открытия Shell вы можете выполнять команды:

```bash
# Проверить версию PHP
php -v

# Проверить, установлена ли Nova
composer show laravel/nova

# Проверить маршруты
php artisan route:list | grep nova

# И другие команды...
```

## Если не можете найти Shell

### Вариант 1: Проверьте права доступа

Убедитесь, что у вас есть права на доступ к сервису.

### Вариант 2: Используйте Deployments

1. Откройте вкладку **"Deployments"**
2. Найдите последний деплой
3. Рядом может быть кнопка для открытия Shell

### Вариант 3: Через настройки

1. Откройте вкладку **"Settings"**
2. Найдите раздел с командами или Shell

## Важно

- Shell открывается для конкретного сервиса
- Убедитесь, что открыли Shell для **Laravel сервиса**, а не MySQL
- Shell работает в реальном времени - вы видите вывод команд сразу

## Скриншоты интерфейса (описание)

**Главная страница проекта:**
- Слева список сервисов
- Справа информация о сервисе
- Вкладки вверху: Deployments, Metrics, Variables, **Shell**, Settings

**После открытия Shell:**
- Внизу страницы или в отдельном окне открывается терминал
- Видна командная строка с приглашением `$`
- Можно вводить команды

## Быстрая проверка

После открытия Shell выполните простую команду:

```bash
pwd
```

Должна показать текущую директорию (обычно `/app`).

Если команда работает - Shell открыт правильно! ✅
